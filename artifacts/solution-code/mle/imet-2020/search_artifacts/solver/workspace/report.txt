# Comprehensive Report: Kaggle-style ML Exploration

## 1. Solutions Explored

During this exploration, multiple approaches were systematically investigated to address noisy, crowdsourced multi-label data. Below is a detailed breakdown of the solutions attempted:

* **Data-Driven Hierarchical Asymmetric Loss (Iteration 10):**
  * **Technique:** Introduced a soft constraint enforcing hierarchical relationships (e.g., if B is present, A must be present). It used standard Asymmetric Loss (ASL) as a base and added a penalty if a child class had a higher predicted probability than its empirical parent class.
  * **Hyperparameters:** `lambda_hier=0.1`, `gamma_neg=4.0`, `gamma_pos=1.0`, `clip=0.05`.
  * **Validation Metric:** 0.6661 F1.
  * **Learned:** Performance degraded relative to previous baselines. The empirical ontological rules proved too rigid and over-regularized the model, suppressing valid non-conforming edge-case predictions.

* **Ablated Asymmetric Loss without Penalty (Iteration 11):**
  * **Technique:** Removed the hierarchical penalty (`lambda=0`) to test standard ASL in isolation.
  * **Hyperparameters:** LR=2e-4, `gamma_neg=4`, `clip=0.05`.
  * **Validation Metric:** 0.6673 F1.
  * **Learned:** Standard ASL improved over the hierarchical loss but remained inferior to earlier Symmetric Focal Loss models. The raw ASL formulation suffered from unoptimized gradient magnitudes.

* **Widened MLP with Focal Loss (Iteration 12):**
  * **Technique:** Swapped back to Focal Loss and attempted scaling the MLP hidden dimensions from 2048 to 4096 and 1024 to 2048, while lowering dropout.
  * **Validation Metric:** 0.6699 F1.
  * **Learned:** Widening the MLP led to suboptimal generalization. The standard 2048->1024 architecture is a better capacity fit for the 3072-dimensional feature space.

* **Focal Loss with Extended Trajectory (Iteration 13):**
  * **Technique:** Reverted to the standard MLP architecture and extended the training trajectory from 120 epochs to 150 epochs.
  * **Validation Metric:** 0.6740 F1.
  * **Learned:** A longer training schedule benefited the Focal Loss, setting a new SOTA and establishing a solid baseline for further loss function tests.

* **Normalized Asymmetric Loss (Iteration 14):**
  * **Technique:** Reintroduced ASL but incorporated a batch-wise normalization of the modulating weights (dividing weights by their mean).
  * **Hyperparameters:** `gamma_neg=2.0`, `gamma_pos=1.0`, `clip=0.05`.
  * **Validation Metric:** 0.6773 F1.
  * **Learned:** Weight normalization stabilized the gradients of ASL. This confirmed the initial hypothesis: ASL excellently handles crowdsourced false negatives via down-weighting easy negatives and margin clipping, but required proper normalization to match Focal Loss's stability.

* **High-Resolution Feature Extraction (Iterations 15 & 16):**
  * **Technique:** Extracted features at a 448x448 resolution instead of 336x336, testing both Normalized Focal Loss (Iter 15) and Normalized ASL (Iter 16).
  * **Validation Metric:** 0.6786 F1 (Focal) and 0.6788 F1 (ASL).
  * **Learned:** Increasing the input resolution allows the frozen DINOv2 backbone to capture higher-density patch tokens, resolving fine-grained visual details. Normalized ASL proved superior at both resolutions.

* **Multi-Resolution Ensemble (Iteration 17 - Best Solution):**
  * **Technique:** Ensembled the predictions of the 10 models trained on 336x336 features with the 10 models trained on 448x448 features.
  * **Validation Metric:** 0.6810 F1.
  * **Learned:** Combining different spatial resolutions provides complementary views. The lower resolution provides broader context per patch, while higher resolution provides detail, culminating in a robust final pipeline.

## 3. Empirical Findings

Throughout the iterative process, several critical insights were developed:

* **What Worked & Why:**
  * **Batch-Normalized Weight Modulation:** Pure ASL suffers from suppressed gradients when negative weights are heavily decayed. By normalizing the dynamic `one_sided_w` vector by its mean, the gradient scale is preserved while maintaining the *relative* down-weighting of easy negatives. This correctly handled missing labels (false negatives) without destabilizing training.
  * **High-Resolution Backbones:** DINOv2 operates natively on patches. Feeding 448x448 images instead of 336x336 yields nearly twice the number of patch tokens, providing a much richer semantic embedding pool.
  * **Ensembling Spatial Resolutions:** Fusing models trained at different resolutions yielded an immediate +0.002 absolute F1 gain. The variance in patch size relative to object size allowed the models to act as highly complementary experts.

* **What Didn't Work & Why:**
  * **Data-Driven Hierarchical Rules:** Attempting to force the model to respect empirical conditional probabilities (e.g., forcing $p_A \geq p_B$) failed. In crowdsourced datasets, seemingly absolute rules have valid edge cases. The soft penalty constrained the model's ability to express nuanced confidence levels, causing widespread score degradation.
  * **Widened Architectures:** Expanding the MLP to 4096 dimensions caused the model to overfit on the cached features, lowering validation F1. A bottlenecked 2048->1024 topology provided superior regularization.

## 4. Best Solution

The final state-of-the-art solution is a Multi-Resolution Normalized ASL Ensemble. It cleanly splits into a heavy feature extraction phase and a lightweight training phase, enabling rapid iteration.

### End-to-End Pipeline
1. **Feature Engineering:** Images are resized and center-cropped to both 336x336 and 448x448 resolutions, then normalized. They are passed through a frozen `vit_giant_patch14_reg4_dinov2.lvd142m` model. The global `[CLS]` token is concatenated with the mean of all spatial patch tokens, producing a rich 3072-dimensional representation per image. These are cached to disk.
2. **Model Definition:** The predictive head is an isolated 3-layer MLP: `Linear(3072, 2048) -> BatchNorm1d -> ReLU -> Dropout(0.2) -> Linear(2048, 1024) -> BatchNorm1d -> ReLU -> Dropout(0.15) -> Linear(1024, 3474)`.
3. **Training Procedure:** Using the cached features, 10 independent models are trained for each resolution. They use AdamW (`lr=2e-3`, `wd=1e-5`) with a CosineAnnealingLR scheduler over 150 epochs.
4. **Objective Function:** The models are supervised using **Normalized Asymmetric Loss**, which dynamically applies a high focal parameter (`gamma_neg=2.0`) and margin clipping (`clip=0.05`) to negative samples to tolerate missing crowd annotations, while maintaining gradient velocity via weight normalization.
5. **Inference & Optimization:** Test-set features are passed through all 20 models (10 at 336x336, 10 at 448x448) and their raw sigmoid outputs are averaged. Finally, a grid search across the validation set isolates the optimal global binarization threshold (0.40) to maximize Micro F1.

### Key Code Snippets

**1. DINOv2 Feature Extractor**
*Why it's important:* By pooling the dynamic spatial patch tokens and concatenating them with the global `[CLS]` token, the model captures both global context and fine-grained local textures without requiring a massive fine-tuning pass over the giant transformer.
```python
class DINOv2Extractor(nn.Module):
    def __init__(self):
        super().__init__()
        self.model = timm.create_model(DINO_MODEL_NAME, pretrained=True, num_classes=0, dynamic_img_size=True)
    def forward(self, x):
        out = self.model.forward_features(x)
        cls_token = out[:, 0]
        patch_tokens = out[:, self.model.num_prefix_tokens:].mean(dim=1)
        return torch.cat([cls_token, patch_tokens], dim=1)
```

**2. Normalized Asymmetric Loss (CombinedLoss)**
*Why it's important:* Standard focal/asymmetric losses can silently crush gradient norms when most labels are easy negatives. The crucial innovation here is detaching the mean of the dynamic weights and dividing by it (`one_sided_w / w_mean`), ensuring stable, high-velocity training even with aggressive negative suppression.
